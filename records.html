ショートカット連携（OCRテキストの取り込み）

iOS のショートカットなどから、マスク済み画像の OCR テキストを送ってこのアカウントの記録に保存できます。 下の「トークンを発行」で出た文字列を、ショートカットの Authorization: Bearer … に使います（ログイン用パスワードではありません）。

発行済みトークン

トークン文字列はセキュリティのため再表示できません。紛失したら削除して新しく発行してください。

    POST の送り方（詳細）
    エンドポイント
    （読み込み後に表示）
    HTTP メソッド
    POST
    認証ヘッダー（必須）
    Authorization: Bearer <発行したトークン>
    例: Authorization: Bearer AbCdEf123...
    Content-Type（必須）
    application/json
    JSON ボディ
    フィールド名必須説明
    fullText または full_text 必須 サーバー OCR と同じ前提のプレーンテキスト（改行は \n）。曲名・難易度・判定数が読み取れる内容であること。
    takenAt または taken_at 任意 スクリーンショットの撮影日時。推奨形式: yyyy/mm/dd hh:mm（例: 2025/03/23 15:30）。秒付き・ISO 8601 も可。タイムゾーン省略時は日本時間 (JST) とみなします。

    リクエスト例（JSON）

    {
  "fullText": "曲名の行\nMASTER\n…",
  "takenAt": "2025/03/23 15:30"
}
    成功時 200
    saved: 記録が新規/更新されたか（既存レコードの上書きルールは Web の OCR 保存と同じ）。
    message: saved: false のとき理由の文字列。
    parsed: サーバーが解釈した曲・難易度・判定・ポイントなど（デバッグ用）。
    400
    full_text が空、または taken_at の形式が解釈できない。
    401
    トークン未指定・無効・削除済み。
    422
    曲が特定できない、判定の合計が合わないなど。レスポンスの detail はオブジェクトで、error（種類）と parsed（中間結果）を含みます。
    503
    サーバーに Node.js が無く、テキストの後処理が実行できない場合。

    記録がまだありません。OCRでリザルトを読み取って保存しましょう。
